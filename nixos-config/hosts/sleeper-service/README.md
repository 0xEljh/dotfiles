## Sleeper Service Profile

This profile is the low-risk first boot target for an in-place Ubuntu -> NixOS migration.

### Flake target

- `.#sleeper-service`

### Required host files

Before building on the target machine, copy the files generated by `nixos-infect`:

```bash
cp /etc/nixos/hardware-configuration.nix ~/nixos-config/hosts/sleeper-service/hardware-configuration.nix
test -f /etc/nixos/networking.nix && cp /etc/nixos/networking.nix ~/nixos-config/hosts/sleeper-service/networking.nix
```

`networking.nix` is optional, but recommended on providers that need static networking.

### Set next boot generation

```bash
source /root/.nix-profile/etc/profile.d/nix.sh
NIX_CONFIG="experimental-features = nix-command flakes" \
  /nix/var/nix/profiles/system/sw/bin/nixos-rebuild boot --flake path:/root/nixos-config#sleeper-service
```

### Web stack (nginx + ACME + app services)

This host now imports:

- `hosts/sleeper-service/services/nginx-acme.nix`
- `hosts/sleeper-service/services/web-apps.nix`

Ad hoc dev/staging hosts:

- `dev-3000.0xeljh.com` -> `127.0.0.1:3000`
- `dev-3001.0xeljh.com` -> `127.0.0.1:3001`
- `dev-5173.0xeljh.com` -> `127.0.0.1:5173`
- `dev-8000.0xeljh.com` -> `127.0.0.1:8000`

These are generic nginx entry points for temporary local servers you start yourself.

Managed websites:

- `0xeljh.com` -> `127.0.0.1:3005`
- `teathegathering.com` -> `127.0.0.1:3006`
- `vamptutor.com` -> `127.0.0.1:3007`

Internal backend:

- `vamp-tutor-backend` listens on `127.0.0.1:18821`

Firewall:

- Public TCP ports: `22` (SSH), `80` (HTTP), `443` (HTTPS)
- App processes bind to `127.0.0.1` and are only exposed through nginx

Managed systemd services:

- `digital-garden`
- `tea-the-gathering`
- `vamp-tutor-website`
- `vamp-tutor-backend`
- `dotfiles-sync` (oneshot job)

Managed timer:

- `dotfiles-sync.timer` runs `~/dotfiles/scripts/run-sync.sh` roughly every 20 minutes.

Service guards:

- Next.js units only start when `<app>/.next/BUILD_ID` exists.
- `vamp-tutor-backend` only starts when `backend/.venv/bin/python` exists.

Deployment workflow per app:

```bash
cd ~/digital-garden && bun install --frozen-lockfile && bun run build
cd ~/tea-the-gathering && bun install --frozen-lockfile && bun run build
cd ~/vamp-tutor-website && bun install --frozen-lockfile && bun run build
cd ~/vamp-tutor-website/backend && uv sync --frozen

sudo systemctl restart digital-garden tea-the-gathering vamp-tutor-website vamp-tutor-backend
```

Useful checks:

```bash
sudo systemctl status nginx digital-garden tea-the-gathering vamp-tutor-website vamp-tutor-backend
sudo journalctl -u nginx -u digital-garden -u tea-the-gathering -u vamp-tutor-website -u vamp-tutor-backend -f
sudo systemctl status dotfiles-sync.timer
sudo journalctl -u dotfiles-sync.service -f
```
