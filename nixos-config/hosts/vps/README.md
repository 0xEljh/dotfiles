## VPS (`sleeper-service`)

In-place Ubuntu -> NixOS migration (via `nixos-infect`), now the main VPS profile.

### Flake target

- `.#sleeper-service` (matches `networking.hostName`, so bare `nixos-rebuild --flake .` also resolves)

### Required host files

Copy the files generated by `nixos-infect` before the first build on a new box:

```bash
cp /etc/nixos/hardware-configuration.nix hosts/vps/hardware-configuration.nix
test -f /etc/nixos/networking.nix && cp /etc/nixos/networking.nix hosts/vps/networking.nix
```

Both files must be tracked by git: flake builds only see tracked files, so the
`builtins.pathExists` guards in `default.nix` silently drop untracked copies.
Strip any empty IPv6 gateway/route stanzas and static `docker0` interface
config that `nixos-infect` emits (see comments in `networking.nix`).

### Landing system updates

Update the lock on the box, commit it, and stage the new generation for the
next boot instead of switching live:

```bash
nix flake update   # or: nix flake lock --update-input nixpkgs
sudo nixos-rebuild boot --flake .#sleeper-service
sudo reboot
```

`nixos-rebuild switch` is blocked whenever the update crosses a
`services.dbus.implementation` change (the system bus cannot be swapped on a
live system) and is risky for large nixpkgs jumps generally. Keep the
provider's VNC console open across the reboot.

### Recovery

- SSH is key-only and no console password is set declaratively. Set a
  break-glass console password once per install with `sudo passwd elijah`
  (it persists; `users.mutableUsers` defaults to true).
- If a new generation comes up without SSH: reboot from the provider panel,
  and in the VNC console pick `NixOS - All configurations` at the GRUB menu,
  then the previous generation. No credentials required.
- After recovering, inspect the failed boot:
  `journalctl -b -1 -p err` and
  `journalctl -b -1 -u network-setup.service -u network-addresses-eth0.service -u sshd.service`.
